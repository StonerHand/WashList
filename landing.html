How matching works

Honest matching.
No silent deletes

Same artist isn't a duplicate. Different songs stay separate. Versions surface — never auto-removed. You decide

/ 01 · Fingerprint

Normalize, then signal

Each track is normalized — case, punctuation, featured-artist tags, parenthetical variants — then fingerprinted: ISRC, title, artist set, duration, metadata cluster

/ 02 · Confidence

Confidence, not certainty

Every match gets 0–100. Above 92 → exact. 75–92 → probable. 50–75 → possible. Below 50 → related version, not a duplicate. You see the score and the reason

/ 03 · Safety

What WashList never does

Same-artist isn't a match. Remixes and live versions stay visible as "related" — never auto-deleted. Nothing leaves your library without you
